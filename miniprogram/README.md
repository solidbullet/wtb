## 开发说明

- **API 基地址**: `http://localhost:8080`（通过 Gateway 访问）
- **金额单位**: 后端返回"分"，前端显示时除以 100
- **品牌色**: 主色 `#E88D2E`、辅色 `#5B8C5A`、背景 `#F9F6F0`

## 页面功能

| 页面 | 功能 |
|------|------|
| 首页 | 公告轮播、快捷入口、推荐菜品（从后台动态获取）、活动预览 |
| 扫码 | 调用微信扫码，解析 seat_id 后跳转到菜单页 |
| 菜单 | 左侧分类导航、右侧菜品列表、底部购物车栏、点击去结算 |
| 菜品详情 | 大图展示、价格/描述、数量选择、加入购物车 |
| 订单确认 | 展示购物车商品、备注输入、提交订单、余额/微信支付 |
| 充值 | 充值套餐（从后台动态获取）、余额显示、促销展示 |
| 我的 | 用户信息、余额/积分/订单快捷入口 |
| 订单 | 全部/待支付/进行中/已完成 订单筛选 |
| 宠物 | 宠物列表展示 |
| 积分 | 积分余额、商品兑换 |
| 活动 | 活动报名、名额进度条 |

## 核心流程

### 点餐流程
1. 首页 / 扫码页 → 获取 `seat_id`
2. 菜单页 / 菜品详情页 → 选择数量 → 加入购物车
3. 菜单页底部购物车栏 → 点击"去结算" → 订单确认页
4. 订单确认页 → 提交订单 → 支付 → 订单完成

### 数据流说明
- **推荐菜品**: 首页调用 `/api/menu/dishes?tags=推荐`，后台按标签查询；若无推荐菜品，自动降级加载全部菜品
- **充值方案**: 调用 `/api/pricing/recharge-plans` 动态获取，支持后台管理
- **购物车**: 基于 `seat_id` 隔离，调用 `/api/order/cart/add|list|remove` 管理
- **订单创建**: 调用 `/api/order/create` 从购物车生成订单，再调用 `/api/pay/create` 创建支付单
