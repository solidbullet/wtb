## 第一章 产品定位与设计原则

### 1.1 产品定位

"汪托帮"是一个宠物主题的线下门店点餐系统，品牌全称 **"汪托帮 · 山水宠物俱乐部"**。用户到店后扫描座位二维码进入小程序，完成浏览菜单、点餐、支付、充值、积分兑换、活动报名等操作。

**一句话描述**：到店扫码 → 点餐支付 → 攒积分兑好礼 → 带宠物参加活动。

### 1.2 用户画像

| 角色 | 场景 | 核心诉求 |
|------|------|----------|
| 宠物主人 | 带狗到店消费 | 快速点餐、会员价、给宠物攒积分 |
| 新顾客 | 首次到店 | 了解菜单、注册即会员 |
| 老顾客 | 高频消费 | 充值优惠、积分兑换、活动报名 |

### 1.3 不在小程序中展示的功能

以下功能属于**后台管理系统**，不在小程序中出现：

- ❌ 营收统计、消费趋势图、仪表盘
- ❌ 菜品销量排行、会员分布分析
- ❌ 订单后台管理（改状态、退单）
- ❌ 座位可视化布局编辑
- ❌ 营销规则配置

### 1.4 设计原则

- **宠物友好**：暖色调为主，大量留白，可爱图标
- **操作极简**：核心路径（扫码→点餐→支付）不超过 4 步
- **一屏一任务**：每屏聚焦一个操作，不堆砌信息
- **即时反馈**：支付成功/积分到账/报名确认，操作后立即弹窗确认

#### 4.1.2 公告轮播 (Slider)

**数据来源**：`GET /api/activity/announcements`

- 取 `status=1` 且当前时间在 `start_time` 和 `end_time` 之间的公告
- 按 `sort_order` 排序
- 每条公告展示图片 + 标题
- 点击跳转：根据 `link_type` 决定
  - `recharge` → 跳转充值优惠 Tab
  - `activity` → 跳转到指定活动详情
  - `external` → 打开 `link_target` 网页
  - `text` → 弹窗展示 `content` 全文
- 自动轮播间隔：4 秒
- 手动可左右滑动
- 指示器：圆点样式，当前高亮

#### 4.1.3 快捷入口

两个等宽的卡片式按钮：

- **扫码点餐**：橙色底 + 白色图标 → 调起微信扫码（`wx.scanCode`）→ 跳转菜单页
- **充值折扣**：绿色底 + 白色图标 → 跳转充值优惠 Tab

#### 4.1.4 近期活动

**数据来源**：`GET /api/activity/list?pageSize=3`

- 最多展示 3 条 `published` 状态的活动
- 每条显示：活动图片、标题、时间、地点、报名进度条
- 报名进度：`current_participants / max_participants`，满额显示"已满"
- 底部"查看全部活动 →"跳转活动列表页

#### 4.1.5 推荐菜品

**数据来源**：`GET /api/menu/dishes?pageSize=6`（取 `tags` 含"推荐"的）

- 横向滚动列表
- 每个菜品卡片：图片、名称、会员价（如有）
- 点击跳转菜品详情

### 4.2 点餐 (Tab 2)

点餐流程是一个**线性流程**，从扫码开始：

```
扫码座位 → 菜品菜单 → 购物车 → 订单确认 → 支付 → 完成
```

#### 4.2.1 扫码页

```
┌─────────────────────────────────────┐
│                                     │
│          [  座位二维码  ]             │
│                                     │
│     ┌───────────────────┐           │
│     │                   │           │
│     │   请扫描桌上的      │           │
│     │   座位二维码        │           │
│     │                   │           │
│     │    [扫码图标]      │           │
│     │                   │           │
│     └───────────────────┘           │
│                                     │
│   📍 扫码后自动识别桌号，              │
│      同座位的朋友可以一起点餐           │
│                                     │
└─────────────────────────────────────┘
```

**交互**：
- 点击扫码区域 → 调起 `wx.scanCode` → 解析二维码内容
- 调用 `GET /api/seat/scan?code=xxx` 获取座位信息
- 成功：跳转菜单页，URL 参数带 `seat_id`
- 失败：弹窗提示"无效的座位二维码"或"该座位已被占用"

- 售罄菜品置灰，点击提示"今日已售罄"

## 第七章 非功能需求

### 7.1 性能

- 首屏加载时间 < 2 秒（4G 网络）
- 图片懒加载 + CDN
- 菜品列表分页加载，每页 20 条
- 购物车操作（加/减/删）即时反馈，无需等待服务端响应（乐观更新）

### 7.2 授权

- 首次打开 → 微信授权登录（`wx.login` 获取 code → `POST /api/user/wx-login`）
- 手机号授权（`getPhoneNumber`）→ 更新用户手机号
- 订阅消息授权（下单时、活动报名时请求授权）

### 7.3 异常处理

| 场景 | 处理方式 |
|------|---------|
| 网络断开 | Toast 提示"网络连接失败，请检查网络" |
| Token 过期 | 自动静默重新登录（`wx.login` 换新 token） |
| 库存不足 | 购物车中对应菜品标记"库存不足"，不可下单 |
| 余额不足 | 支付时弹窗提示 + "去充值"按钮 |
| 座位被释放 | 提示"座位已过期，请重新扫码" |

### 7.4 多人同步

同一座位多人同时点餐时的购物车同步策略：

- 购物车存储在 Redis（`cart:{seat_id}`）
- 每次操作后，前端定时轮询刷新（或使用 WebSocket）
- 简化方案：打开购物车时重新拉取 `GET /api/order/cart/list`

## 第八章 前端工程实现（给 Kimi Code）

### 8.1 HTML 原型文件

开发者提供 HTML 设计原型文件，放在以下目录：

```
miniprogram/prototypes/
├── home.html      ← 首页设计原型
├── mine.html      ← 「我的」页面设计原型
└── ...            ← 后续补充的页面原型
```

### 8.2 转化流程

Kimi Code 需要完成以下工作：

1. **读取 `prototypes/` 下所有 HTML 文件**，理解页面结构和样式
2. **提取 CSS**：扫描所有 `<style>` 块 →
   - 跨页面重复的规则 → 移入 `app.wxss`（全局样式）
   - 页面独有规则 → 保留在对应 `.wxss`
3. **转化 HTML → WXML**：标签映射（`div→view`, `span→text`, `img→image`, `onclick→bindtap`），静态数据替换为 `{{}}` 绑定
4. **编写 JS**：数据绑定、API 调用（封装到 `utils/api.js`）、事件处理
5. **配置 `app.json`**：注册所有页面路径 + 4 Tab 导航栏

### 8.3 目录结构

```
miniprogram/
├── prototypes/          ← HTML 原型（你提供）
├── app.json             ← Kimi Code 生成
├── app.wxss             ← Kimi Code 生成（共享 CSS）
├── app.js               ← Kimi Code 生成
├── pages/               ← Kimi Code 生成
│   ├── index/           ← 首页 (← home.html)
│   ├── order/           ← 点餐（扫码/菜单/购物车/确认/结果）
│   ├── recharge/        ← 充值优惠
│   ├── mine/            ← 我的 (← mine.html) + 子页
│   ├── points/          ← 积分商城主页 + 流水
│   └── activity/        ← 活动列表 + 详情 + 报名
├── components/          ← 可复用组件
├── utils/               ← api.js / auth.js / util.js
└── README.md            ← 详细开发指南
```

### 8.4 关键约定

- **类名不变**：你的 HTML 中定义的所有 class 名称直接保留到 WXML/WXSS
- **品牌色**：主色 `#E88D2E`、辅色 `#5B8C5A`、背景 `#F9F6F0`——已在 `app.wxss` 中预定义为全局样式类
- **金额单位**：后端返回"分"，前端显示时除以 100（`{{item.price / 100}}`）
- **API 调用**：所有后端接口参考 `API_SPEC.md`，请求/响应格式完全匹配
- **页面路由**：使用 `页面清单`（附录A）中的路径——`<navigator>` 和 `wx.navigateTo` 保持一致

详细的前端开发指南见 `miniprogram/README.md`。
